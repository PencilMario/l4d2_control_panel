# 私有覆盖文件管理器与控制台跟随设计

日期：2026-07-15

## 1. 目标

将现有简易“实例私有覆盖”表单升级为实例详情中的独立“私有文件”Tab，使管理员能完整管理 `instances/<id>/private/` 文件树。文件操作先暂存，只有点击“应用更改”才通过后台 Job 同步到 `game/left4dead2/`。

同时修正原生控制台滚动行为：进入控制台或提交指令时滚到底部并启用“跟随最新输出”；仅在跟随开启时随 WebSocket 新输出继续滚动。用户上翻时关闭跟随，手动回到底部时恢复跟随。

## 2. 范围与非目标

范围包括：

- 只读写 `instances/<id>/private/`，不浏览或直接编辑 `game/left4dead2/`。
- 目录树、目录列表、上传、下载、新建文件、新建目录、在线文本编辑、重命名、移动、删除、历史快照、差异预览、整体回滚和批量应用。
- 已应用清单、未应用变更和最近 20 次应用快照。
- 私有覆盖撤销后恢复正确的下层来源。
- 控制台跟随状态及其滚动边界。

非目标包括：

- 任意 Shell、脚本执行、宿主机文件浏览或游戏目录文件管理。
- 在线编辑二进制文件或超出文本编辑上限的大文件。
- 多选批量文件操作、跨实例复制、文件搜索和协同编辑。
- 改变插件包、共享 VPK 或 Valve 文件的既有所有权和更新策略。

## 3. 既有约束

覆盖优先级保持为：

```text
插件包 < 共享 VPK < 实例私有覆盖层
```

现有 `PrivateManager` 已支持安全相对路径、文本保存、读取、平面列表、单文件历史、删除和整树复制，但当前保存会立即排队应用，删除也不会从已部署目录撤销旧覆盖。本设计以一个已应用清单取代“只复制当前树”的判定方式，不增加第二套部署所有者。

应用操作继续进入实例现有的持久化 Job 队列，与更新、重配等变更串行。Panel 不执行管理员上传内容。

## 4. 方案选择

采用清单驱动的暂存工作区：

1. 文件 CRUD 立即、原子地更新 `private/` 源树，但不触碰游戏目录。
2. Panel 将当前树与上次成功应用清单比较，向 UI 返回新增、修改、移动和删除差异。
3. 管理员点击“应用更改”后，Job 先创建当前应用快照，再按差异更新游戏目录。
4. Job 全部成功后才原子更新已应用清单并清除未应用状态。
5. Job 失败时保留原已应用清单、当前暂存树和诊断信息，允许修正后重试。

直接写游戏目录会破坏批量确认和失败一致性；每次整树打包重放会放大 I/O 且不能可靠表达逐文件撤销，因此不采用。

## 5. 文件管理交互

“私有文件”作为实例详情独立 Tab。桌面端采用已确认的 A 布局：左侧为可展开目录树，右侧为当前目录列表或内嵌文本编辑器。窄屏下目录树进入抽屉，主内容保持全宽。

顶部工具栏提供上传文件、新建文件、新建目录和刷新。文件或目录的上下文菜单按能力提供下载、编辑、重命名、移动、查看历史和删除。二进制文件只允许下载、替换、移动、重命名或删除；只有限定大小且可解码为 UTF-8 的普通文件可在线编辑。

固定状态栏显示新增、修改、移动和删除数量，并提供“查看差异”和“应用更改”。没有差异时禁用应用按钮。删除目录、覆盖同名目标和整体回滚要求确认。

应用成功后刷新树、清空未应用状态并增加快照；失败后保持编辑现场和未应用差异。空目录是受支持的资源，必须在清单与快照中保留。

## 6. 文件操作与应用数据流

文件 API 使用规范化、URL 编码的相对路径，覆盖以下能力：

- 查询目录树和未应用差异。
- 分块上传及原子完成；小文本直接读取和保存。
- 创建目录、移动或重命名文件/目录、递归删除。
- 下载文件。
- 查询应用快照、快照差异和发起整体回滚。
- 发起应用 Job。

已应用清单记录路径、类型、内容哈希、大小、权限策略和下层来源恢复所需的所有权信息。移动可在 API 层表达为一个操作，在差异层以旧路径删除和新路径新增处理，保持部署逻辑单一。

撤销一个已应用的私有文件时，Job 不只删除 `private/` 源文件对应的游戏副本。它根据内容所有权清单恢复被遮蔽的共享 VPK、插件包或 Valve 基础文件；若下层不存在，则删除目标。共享 VPK 符号链接仍由既有受控同步逻辑创建，私有管理器不得自行接受符号链接。

## 7. 快照与回滚

每次成功应用对应一个不可变快照，包含完整私有树清单和恢复内容所需的数据。编辑、上传等暂存动作不单独生成历史版本。

每个实例默认保留最近 20 个成功应用快照。创建第 21 个快照后，在新应用已成功且当前快照不再需要回滚保护时清理最旧快照。清理失败只留下可诊断告警，不把已经成功的应用改判为失败。

整体回滚会把选定快照恢复为新的暂存树，先展示差异，再由管理员发起新的应用 Job。它产生一条新快照，不改写或删除历史。

## 8. 控制台跟随状态

控制台组件持有布尔状态 `followLatest`，默认关闭。行为定义如下：

- 打开或重新进入控制台：在输出已渲染后滚到底部，并设置 `followLatest = true`。
- 提交控制台指令：立即滚到底部并设置 `followLatest = true`。
- WebSocket 回放或新增输出：只有 `followLatest` 为真时，在 DOM 更新后滚到底部。
- 用户向上滚离底部容差区：设置 `followLatest = false`，后续输出不改变当前位置。
- 用户手动滚回底部容差区：设置 `followLatest = true`，后续输出继续滚动。

底部判断允许少量像素容差，避免缩放和亚像素取整导致状态抖动。程序自身滚动不得被误判为用户上翻。断线重连不强制改变用户已关闭的跟随状态；只有重新进入控制台或提交指令会强制开启。

## 9. 安全与错误处理

- 实例 ID 和所有路径必须通过安全路径连接；拒绝绝对路径、`..`、空段、平台保留名称、NUL 和符号链接逃逸。
- 上传写入实例内临时区域，校验单文件大小、实例总容量和可用磁盘后原子移动；中断上传可清理或恢复，不暴露半文件。
- 在线编辑有独立字节上限并要求 UTF-8；下载和上传不受文本编辑上限约束，但受上传配额约束。
- 移动和重命名不得静默覆盖；目标已存在时必须由显式确认后的请求表达。
- 应用清单仅在 Job 成功后原子替换。Panel 中断或 Job 失败时，恢复旧游戏目录状态；若恢复也失败，Job 标记失败并保留明确诊断与修复入口。
- 所有变更、应用和回滚进入现有审计记录。

## 10. 兼容与迁移

已有 `private/` 文件原样成为暂存工作区内容，不移动路径。首次启用清单时，以当前游戏目录中的既有私有应用状态建立基线；不得把所有现有文件误报为待应用，也不得删除无法确认所有权的文件。

现有内容页面中的简易私有文件列表和“保存并立即应用”表单由实例详情 Tab 取代。插件包、共享 VPK、更新流水线、持久化挂载、Host 网络和 PTY 控制台协议保持不变。

## 11. 测试与验收

后端测试覆盖：

- 路径穿越、符号链接、非法实例 ID、配额与磁盘不足。
- 文件和目录 CRUD、空目录、移动冲突、分块上传中断及原子完成。
- 当前树与已应用清单的差异计算。
- 应用成功、应用失败回滚、失败后重试和 Job 串行化。
- 删除私有覆盖后分别恢复插件包、共享 VPK、Valve 文件或空目标。
- 20 个快照保留、清理失败告警、差异查看和整体回滚产生新历史。

前端单元与浏览器测试覆盖：

- 独立 Tab 的目录树、目录列表、文本/二进制能力、未应用状态、确认流程和错误恢复。
- 桌面两栏布局与移动端抽屉布局。
- 上传、编辑、移动、删除、应用、刷新恢复和快照回滚主流程。
- 控制台进入和提交指令强制开启跟随；跟随开启时新增输出滚动；用户上翻后新增输出不滚动；手动回底后恢复跟随；断线重连尊重已关闭状态。

验收标准是管理员能只通过该 Tab 完成 `private/` 整棵树的安全管理，批量应用或撤销覆盖，并在失败后无数据丢失地重试；控制台在用户查看历史输出时不抢走滚动位置，在用户位于底部时持续跟随最新输出。

## 12. 设计输入与影响边界

### TaskIntentDraft

管理员需要完整而独立的实例私有覆盖文件管理体验，并需要符合终端习惯的控制台滚动。高风险点是路径安全、撤销覆盖时的下层恢复、应用原子性、历史容量和用户滚动意图识别。

### BaselineReadSetHint

设计以 `docs/aegis/specs/2026-07-14-l4d2-control-panel-design.md` 的私有覆盖、内容所有权、安全和页面结构为基础，并核对当前 `internal/content/private.go`、HTTP 路由、React 内容页、控制台组件及现有 Go/Vitest/Playwright 测试。实现计划仍需逐项确认现有内容部署所有权和数据库迁移入口。

### ImpactStatementDraft

受影响层包括私有内容服务、持久化清单/快照、HTTP API、实例 Job 流水线、React 实例详情导航与文件管理组件、控制台滚动状态以及端到端测试。兼容边界是保留现有目录、覆盖优先级、Job 串行化、PTY 通道和安全模型；不开放游戏目录或任意命令执行。
