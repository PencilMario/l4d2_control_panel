# 私有文件 ZIP 导入导出设计

日期：2026-07-16

## 1. 目标

在现有“私有文件”页面为每个游戏实例增加整棵私有文件工作区的 ZIP 导入和导出能力。

导入采用完全替换语义：ZIP 中的目录树成为新的私有文件工作区，任何未出现在 ZIP 中的当前文件、空目录和未应用修改都不会合并或保留在新工作区中。导入仅产生待应用差异，不自动修改 `game/left4dead2/`；管理员仍须检查差异并点击“应用更改”。

## 2. 范围与非目标

范围包括：

- 从所选实例的当前私有文件工作区生成 ZIP。
- 通过 ZIP 原子替换所选实例的当前私有文件工作区。
- 保留 ZIP 内的相对路径、唯一顶层目录和显式空目录。
- 导入前的破坏性操作说明与确认。
- 归档安全校验、容量限制、失败回滚、审计和自动化测试。

非目标包括：

- 导入后自动应用私有文件。
- 合并导入、冲突选择或只导入部分文件。
- 在 ZIP 中包含已应用清单、历史快照、差异或其他 Panel 元数据。
- 删除或改写既有历史应用快照。
- 自动剥离唯一顶层目录、忽略 `__MACOSX` 等目录或改写归档路径。
- 导入 TAR、7z、RAR 或其他归档格式。

## 3. 已确认语义

“私有文件工作区”指所选游戏实例 `instances/<id>/private/` 所代表的暂存目录树。它与已部署游戏目录和历史应用快照不同。

导入成功后的工作区只包含 ZIP 中通过校验的普通文件和目录。旧工作区中不在 ZIP 内的内容从当前工作区消失，包括此前尚未应用的内容。历史应用快照和上次成功应用清单保持不变，因此现有差异计算会把新工作区与已应用版本比较，并正确显示新增、修改和删除。

ZIP 中只有一个顶层目录时仍保留该目录。导出内容位于归档根路径；将 Panel 导出的 ZIP 再导入可恢复同样的工作区目录结构。

## 4. 方案选择

采用服务端“校验并暂存解压，然后事务替换”的方案：

1. HTTP 层接收 ZIP，并限制压缩数据大小。
2. 内容层完整检查归档，将其解压到实例目录内的临时工作区。
3. 只有全部条目写入、同步和复核成功后，才在实例写锁内替换当前工作区。
4. 替换失败时恢复旧工作区；进程中断时由同一事务记录清理或恢复。
5. 导入完成后不修改已应用清单，也不创建应用快照。

不采用浏览器解压后逐项调用文件 API，因为它无法保证整棵树的原子性，且中途失败会留下混合状态。不采用直接向当前工作区解压，因为校验或写入失败会破坏原内容。

该方案复用 `PrivateManager` 的实例级写锁、安全路径和工作区替换惯例，保持私有工作区只有一个写入所有者。它是既有内容层能力的局部扩展，不建立新的部署所有者，因此不需要独立 ADR。

## 5. API 与组件边界

HTTP API 增加同一资源的两个操作：

- `GET /api/instances/{id}/private/archive`：流式返回当前工作区 ZIP，并设置附件文件名。
- `POST /api/instances/{id}/private/archive?confirm=true`：以 `application/zip` 接收归档并完全替换工作区。缺少确认参数返回 `428 confirmation_required`。

HTTP 层负责鉴权、实例存在性、内容类型、压缩数据读取上限、确认前置条件、下载响应头和错误码映射。归档遍历、校验、暂存、替换与恢复由 `PrivateManager` 负责，使其他调用者不能绕过内容层不变量。

导出只读取当前工作区。导出期间持有与现有文件操作兼容的实例读边界，避免一个 ZIP 同时包含修改前后的状态。空工作区导出为有效的空 ZIP。

## 6. 导入事务与安全

导入归档只接受普通文件和目录，并执行以下校验：

- 拒绝绝对路径、盘符路径、反斜线歧义、空路径、NUL、`.`、`..` 和安全根目录外路径。
- 拒绝符号链接、其他特殊文件、加密条目和不受支持的 ZIP 特性。
- 路径按最终规范化结果检查重复、大小写碰撞和文件/目录冲突。
- 保留显式空目录，但不得让文件成为另一条目的父目录。
- 限制压缩数据大小、展开总大小、单文件大小、文件数量和压缩比，避免 ZIP 炸弹与磁盘耗尽。
- 不跟随现有或暂存路径中的符号链接；临时目录必须位于受控实例根内。
- 解压结束后重新扫描暂存树，确认其清单与校验结果一致。

实例写锁覆盖最终替换，并与保存、上传完成、移动、删除、快照恢复和差异读取的既有同步边界协调。替换过程使用同级暂存目录和恢复记录：先准备完整新树，再切换工作区；任何对外成功响应都意味着新树已成为唯一当前工作区。失败响应必须保持或恢复导入前可见工作区，不得暴露半棵新树。

旧工作区仅可作为事务恢复材料短暂存在，不能与新树合并，也不能通过私有文件 API 访问。事务完成后清理恢复材料；中断恢复沿用内容层启动恢复机制。

## 7. 用户交互与说明

“私有文件工具栏”新增“导入 ZIP”和“导出 ZIP”。导出按钮直接下载所选实例的当前工作区；导入按钮只接受 `.zip` 文件。

选中文件后、上传前必须显示包含实例和文件名的确认提示，核心文案为：

> 导入会完全替换当前私有文件工作区。ZIP 中不存在的现有文件和未应用更改将被删除，不会保留；历史应用快照不受影响。导入后不会自动应用到游戏目录。

取消确认时不发送请求。导入期间禁用会改变所选实例工作区的操作，并显示进行中状态。成功后清除失效的文件选择和编辑状态，刷新目录树、差异及快照列表，并显示：

> 工作区已完全替换，请检查差异后应用更改。

导入错误显示可操作的中文错误，文件树和当前选择保持导入前状态。导出失败走页面既有错误区域，不触发空白或错误 ZIP 下载。

## 8. 错误与审计

客户端错误使用稳定错误类型区分：未确认、非 ZIP 或错误内容类型、无效归档路径、归档结构冲突、归档超限和不支持条目。I/O、同步或恢复失败作为服务端错误记录完整诊断，但响应不暴露宿主机绝对路径。

导入作为一次工作区替换写入现有审计日志，目标包含实例而不包含宿主机路径；结果记录成功或稳定错误码。导出是只读操作，沿用现有只读 API 审计策略。

## 9. 兼容边界

现有单文件上传、文本保存、目录操作、差异、历史快照和“应用更改”协议保持不变。导入不修改私有覆盖优先级、下层恢复、Job 串行化或游戏部署流程。

既有已应用清单是导入后差异的比较基线。历史快照仍可恢复为新的暂存工作区；该恢复会再次完全替换当前工作区，但与 ZIP 导入是两个独立入口。归档不携带文件权限策略，导入文件和目录使用既有私有工作区的安全默认权限。

## 10. 测试与验收

内容层测试覆盖：

- 导出并重新导入普通文件、嵌套目录、唯一顶层目录、空目录和空工作区。
- 导入新树后，旧文件和旧空目录不在当前工作区中。
- 已应用清单和历史快照不变，差异正确表达导入后的新增、修改与删除。
- 无效 ZIP、路径穿越、绝对路径、反斜线、符号链接、加密条目、重复路径、大小写碰撞、文件/目录冲突及各类容量超限。
- 校验、解压、同步、替换和清理故障下不暴露部分新树，并恢复旧工作区。
- 与并发保存、删除、上传完成、快照恢复及导出的实例级同步。

HTTP 测试覆盖下载头和 ZIP 内容、确认前置条件、内容类型、读取上限、稳定错误码、实例不存在及审计结果。

前端测试覆盖按钮可用性、ZIP 文件选择、完整警告文案、取消不请求、成功后刷新与状态提示、失败后保留现有视图、导出下载以及切换实例时请求归属正确。Playwright 主流程覆盖导出工作区、修改工作区、再导入并确认完全替换但尚未应用。

验收标准是：管理员能够导出所选实例的当前私有文件目录树；导入任意合法 ZIP 后，当前工作区与 ZIP 精确对应且不混入旧文件；导入失败不改变旧工作区；导入成功只产生待应用差异，并明确引导管理员检查后手动应用。

## 11. 设计输入与影响边界

### TaskIntentDraft

为“私有文件”页面增加整棵工作区的 ZIP 导入导出。导入必须完全替换而非合并，并向管理员明确说明未出现在 ZIP 中的现有文件和未应用更改不会保留。导入保持暂存语义，不自动部署。

### BaselineReadSetHint

设计以 `CONTEXT.md`、`docs/aegis/specs/2026-07-15-private-file-manager-console-follow-design.md`、`internal/content/private.go`、`internal/content/private_state.go`、`internal/archive/inspect.go`、`internal/httpapi/server.go`、`web/src/app/PrivateFilesPage.tsx` 及对应 Go、Vitest、Playwright 测试为依据。实施时需要核对现有内容大小配置和私有管理器启动恢复入口，以决定具体限制注入点和恢复记录复用方式。

### ImpactStatementDraft

受影响层是归档校验、私有内容服务、HTTP API、React 私有文件页面及自动化测试。必须保持的边界包括：私有工作区是唯一导入目标；已应用清单和快照不属于 ZIP；导入不触碰游戏目录；所有路径受安全根约束；失败时旧工作区仍完整可见。非目标是合并导入、自动应用、跨实例导入和新增归档格式。
