# 前端动效与异步操作防重复设计

## 目标

在不改变控制面板现有工业深色风格、信息架构和业务流程的前提下，为常用组件补充舒适、克制且一致的交互动效，并确保所有异步写操作在执行期间不能被重复触发。

采用“层次增强”动效方向：交互反馈比当前界面更明确，但不使用会干扰长时间运维操作的大幅缩放、持续发光或戏剧化转场。

## 视觉与动效

- 卡片、导航、表格行、工具按钮和图标使用统一的 160 至 240 毫秒缓动。
- 可交互卡片在悬停时轻微抬升，并增强边框、背景和阴影层次；组件稳定尺寸不变。
- 按钮提供悬停、按下、焦点、禁用和忙碌反馈。图标仅做小幅位移或旋转，不推动相邻文本。
- 页面主要内容、弹窗和列表项可使用轻微透明度与纵向位移动画；列表顺序动画保持短促，不延迟用户操作。
- 所有动效遵循 `prefers-reduced-motion: reduce`。启用减少动态效果时，取消非必要动画、平滑滚动和变换。
- 保留现有颜色、字体、布局密度和组件层级，不引入新的动画运行时依赖。

## 异步写操作

- 登录、游戏实例启停和更新、实例配置保存、内容保存或删除、文件上传与应用、计划任务保存或删除、设置保存、玩家踢出或封禁等写操作必须具有本地忙碌状态。
- 第一次触发后立即设置忙碌状态；在请求完成前，事件处理函数必须拒绝后续触发，对应控件同时设置 `disabled`。
- 忙碌按钮保持稳定宽高，优先保留原标签语义，并显示旋转图标或“处理中”“保存中”等明确状态。
- 忙碌控件设置适当的 `aria-busy` 或可访问状态文本。图标按钮仍保留可辨识的 `aria-label`。
- 锁定粒度限定到当前操作或当前资源。操作游戏实例 A 不应阻止操作游戏实例 B；同一弹窗内互斥的提交操作可以共享锁。
- 只读请求、页面导航、弹窗开关、选项切换和普通数据刷新不纳入写操作锁。已有读取加载态继续沿用现有行为。

## 状态与错误处理

- 现有 `submitting`、`busy`、`deleting`、`savingJobs` 等局部状态继续作为各自页面的行为所有者。
- 缺少防重复状态的入口在所属组件补齐，不建立会冻结整个应用的全局请求锁。
- 请求成功后沿用现有刷新、轮询、关闭弹窗或提示流程。
- 请求失败时恢复可操作状态，并保留现有错误提示；异常和同步抛错都必须通过 `finally` 释放锁。
- 后台任务创建成功后即可释放“创建任务”的提交锁；后台任务自身的生命周期继续由现有任务轮询与任务页面表达。

## 组件边界

- 共享 CSS 负责统一视觉状态、缓动曲线、忙碌图标和减少动态效果规则。
- React 组件继续拥有业务忙碌状态，避免通用视觉组件接管请求或业务错误处理。
- 仅在多个入口确实重复同一套展示逻辑时提取轻量按钮辅助组件；本次不要求一次性迁移所有按钮到新的组件体系。

## 兼容边界与非目标

- 不改变 HTTP API、请求载荷、任务模型、路由或后端幂等语义。
- 不改变页面布局、导航结构、品牌配色或现有确认流程。
- 不使用全页遮罩阻止无关操作，不把所有请求串行化。
- 不为纯读取操作增加不必要的禁用行为。
- 不引入 Motion 等第三方动画库。

## 验证

- 组件测试先覆盖登录和游戏实例写操作：快速重复点击只发出一次请求，完成或失败后恢复按钮。
- 补充代表性页面测试，确认保存、删除、上传或应用操作在忙碌期间禁用。
- 测试不同游戏实例的操作锁互不影响。
- 样式检查覆盖悬停、焦点、禁用、忙碌和 `prefers-reduced-motion` 规则。
- 运行完整前端单元测试、TypeScript 构建和 Playwright 主流程。
- 在桌面和移动视口检查按钮尺寸稳定、文字不溢出、弹窗不重叠、悬停与焦点反馈清晰。

- BaselineReadSetHint：`CONTEXT.md`、`web/src/app/App.tsx`、各独立页面组件、`web/src/styles/app.css`、现有 Vitest 与 Playwright 测试。
