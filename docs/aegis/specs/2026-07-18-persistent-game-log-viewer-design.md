# 持久游戏日志查看与保留设计

- 状态：已批准
- 日期：2026-07-18
- 范围：实例持久日志、日志查看 API、实例日志页面、清理任务与系统设置

## 1. 目标

管理员应能按当前游戏实例查看 Left 4 Dead 2 游戏日志与 SourceMod 日志。SourceMod 日志允许包含任意层级的子目录。查看器提供文件树、只读文本预览、刷新和原文件下载，并对常见日志语义和 ANSI 颜色进行高亮。

日志属于实例持久数据：游戏重装、插件重装、容器重建和 OverlayFS 重建不得删除尚在保留期内的日志。日志默认保留 14 天，使用全局设置调整。过期清理由现有后台任务系统执行，结果在任务流水和完整任务日志中可见。

## 2. 范围与非目标

范围包括：

- 每个实例独立持久保存游戏日志和 SourceMod 日志。
- 将持久目录挂载到游戏与 SourceMod 的标准日志路径。
- 升级现有实例时迁移 Overlay 写层中的既有日志。
- 浏览日志文件树、预览文本、刷新和下载原文件。
- 语义与 ANSI 颜色高亮。
- 全局保留天数设置、每日自动清理和立即清理。
- 清理任务去重、进度与结果记录。

非目标包括：

- 在线编辑、删除或重命名日志。
- 跨文件全文搜索、实时跟随或合并多个日志流。
- 把日志内容写入 Panel 数据库、私有文件快照或实例配置导出。
- 永久归档、异地备份或在实例永久删除后保留日志。
- 解析日志并生成统计报表或告警规则。

## 3. 持久目录与运行时挂载

每个实例增加两个由 Panel 管理的持久目录：

```text
instances/<id>/logs/game/
instances/<id>/logs/sourcemod/
```

游戏容器将它们分别挂载到：

```text
/opt/l4d2/game/left4dead2/logs
/opt/l4d2/game/left4dead2/addons/sourcemod/logs
```

挂载点覆盖 Overlay 合并视图中的对应目录，使游戏和 SourceMod 直接写入持久目录，不采用事后复制。日志所有权因此与游戏本体、插件包和 Overlay 写层解耦。普通重装和重建只更新其原有所有者管理的内容，不触碰 `instances/<id>/logs/`。

实例永久删除继续删除整个实例目录，因此会删除日志。该行为必须沿用实例永久删除的显式确认与任务语义。日志不进入私有覆盖快照、插件包、共享 VPK、游戏本体发布版本或实例配置导出。

## 4. 既有日志迁移

现有实例首次启用持久挂载前，Panel 检查 Overlay 合并目录与写层中的两个标准日志位置，把可确认属于该实例的普通日志文件迁移到持久目录。SourceMod 子目录结构保持不变。

迁移不得静默覆盖同名文件。目标不存在时保留原相对路径；目标已存在且内容相同时跳过重复副本；内容不同时为迁入文件增加包含迁移时间和稳定序号的后缀。迁移只接受受控根目录内的普通文件和目录，拒绝符号链接及其他特殊文件。

目录准备、迁移和容器挂载必须按顺序完成。迁移失败时不得启动使用新挂载契约的容器，避免一部分日志留在旧路径、一部分写入新路径。迁移成功后可重复执行，不重复复制同一内容。

## 5. 查看服务与 HTTP 契约

后端新增实例日志服务，唯一允许访问的根是当前实例的 `logs/game` 和 `logs/sourcemod`。所有实例 ID、日志类别和相对路径经过既有安全路径规则；拒绝绝对路径、`..`、NUL、符号链接逃逸和非普通文件。

认证后的实例 API 提供：

- 查询两类日志的递归文件树，包括相对路径、大小和最后修改时间。
- 读取单个日志的文本预览。
- 下载单个日志的原始内容。

文本预览最多返回文件末尾 10 MiB，并返回是否截断、原文件大小和读取时观察到的最后修改时间。读取末尾更符合诊断最新问题的主要场景。非 UTF-8 字节使用替换字符显示，不改变原始文件。下载不受文本预览上限约束，并使用安全文件名与附件响应头。

文件可能在列举、预览或下载期间被游戏轮转。文件不存在时返回明确的“日志已轮转或删除”；文件大小或修改时间变化不被当作服务器错误，刷新后重新读取即可。实现不得因为查看而修改访问时间之外的业务元数据，也不得延长保留期。

## 6. 日志查看页面

实例详情增加独立“游戏日志”入口。桌面端复用私有文件页面已经验证的两栏信息结构：左侧是按“游戏日志”和“SourceMod 日志”分组的递归文件树，右侧是只读日志查看器。窄屏下文件树进入抽屉，查看器保持全宽。

工具栏提供刷新和下载。选择文件后展示路径、大小、最后修改时间与截断提示。加载、空目录、文件轮转、权限失败和下载失败均在局部显示，不清空仍有效的文件树。

前端采用专用日志 tokenizer 与 ANSI 渲染依赖，而非引入完整代码编辑器。逐行识别并着色：

- ANSI 前景色与强调样式；
- 日期、时间和时间戳；
- INFO、WARN、ERROR 等日志等级；
- SourceMod 插件或模块名；
- 玩家名、SteamID 和 UserID；
- IPv4、IPv6 与端口；
- 异常、调用栈和文件行号。

无法识别的文本按原样显示。渲染前必须转义日志内容，ANSI 与语义高亮只生成受控样式 token，不允许日志注入 HTML。查看器使用等宽字体、保留空白和换行，并允许横向滚动长行。

## 7. 保留设置与清理任务

普通系统设置新增 `game_log_retention_days`，缺少记录时为 14，允许整数 1 至 365。系统设置页面增加“游戏日志”区域，展示保留规则并提供保存和“立即清理”按钮。

清理使用新的实例任务类型 `cleanup_game_logs`，与该实例其他变更遵守现有任务串行规则。任务递归扫描两个持久日志根目录，以文件最后修改时间早于任务启动时计算的截止时间作为过期条件。任务只删除普通文件；随后自底向上移除空子目录，但永不删除两个日志根目录。

任务日志至少记录保留天数、截止时间、扫描文件数、过期文件数、成功删除数、跳过数、释放字节数和失败摘要。清理继续处理彼此独立的文件；任一应删除文件失败时，任务最终标记失败并保留具体错误，其余已成功删除的文件不回滚。

调度器每天为每个实例创建一次清理任务。管理员调低保留天数并成功保存后，为所有实例立即排队清理；调高时不恢复已删除日志，也无需立即清理。“立即清理”按当前设置为所有实例排队任务。

同一实例已有 pending 或 running 的 `cleanup_game_logs` 时，不再创建重复任务。批量排队对各实例独立：某个实例排队失败不阻止其他实例，接口返回成功、去重和失败统计。保存设置与排队清理分开提交；设置持久化成功后即生效，排队失败通过响应和 Panel 诊断日志暴露，避免任务系统短暂故障回滚有效设置。

## 8. 生命周期与兼容边界

现有游戏路径、插件包、共享 VPK、私有覆盖优先级和 OverlayFS 发布机制保持不变。新增日志目录挂载只接管两个标准日志路径的写入所有权。

容器重建、选择性游戏重装、插件重装、游戏本体更新和 Overlay 重置必须保留 `instances/<id>/logs/`。实例永久删除仍按现有确认流程删除整个实例，包括日志。迁移完成后，不再把 Overlay 中的旧标准日志路径视为日志所有者。

清理任务是唯一基于保留期自动删除日志的机制。查看 API 不提供修改能力，系统设置请求不直接删除文件，Panel 启动也不在请求外同步删除；启动后由调度器按既有恢复与补调度语义创建任务。

## 9. 故障与恢复

- 持久日志目录无法创建或挂载时，实例启动失败并报告明确诊断，不能退回写入易丢失的 Overlay 路径。
- 既有日志迁移失败时保持旧数据，不启用新挂载；重试应幂等。
- 日志文件在预览期间增长、截断或轮转时返回一个安全快照或明确的已轮转结果，不缓存半个响应供后续复用。
- 单个目录或文件无权限时，文件树和清理任务记录具体受影响路径；受控根之外的信息不回传。
- 自动清理排队失败不会影响实例运行，下一次每日调度或手动立即清理可重试。
- 清理任务中部分删除成功、部分失败时，以失败终态保留诊断；已删除文件不恢复。
- 高亮解析异常只降级为纯文本，不阻止查看和下载。

## 10. 测试策略

后端测试覆盖：

- 容器规格包含两个持久日志挂载，重装与 Overlay 重建不删除日志。
- 新实例目录创建、旧实例日志迁移、同名同内容去重、同名异内容保留和迁移幂等。
- 实例 ID、类别、相对路径、符号链接和特殊文件的安全边界。
- 递归文件树、SourceMod 子目录、末尾 10 MiB 截断、非 UTF-8 预览、原文件下载和轮转竞态。
- 默认 14 天、1 与 365 边界、非法设置拒绝。
- 截止时间边界、递归清理、空目录移除、根目录保留、部分失败统计和释放空间统计。
- 每日排队、设置降低后排队、立即清理、同实例任务去重和批量部分失败。
- 清理任务进度、终态和完整任务日志可见。

前端测试覆盖：

- 两类文件树、SourceMod 嵌套目录、文件选择、刷新、下载及响应式抽屉。
- 加载、空目录、截断、轮转、权限和下载错误状态。
- ANSI、时间、等级、模块、玩家/SteamID、IP 和堆栈高亮，以及 HTML 注入转义和纯文本降级。
- 设置加载、边界校验、保存、调低后的排队反馈和立即清理去重反馈。

端到端验收覆盖：创建两类日志并从实例页面查看高亮预览和下载；执行插件或游戏重装后日志仍存在；把文件时间设置为保留期外，发起立即清理并在任务流水看到统计，随后确认文件消失而新日志保留。

回归验证包括全部 Go 测试、`go vet ./...`、前端单元测试、生产构建和相关 Playwright 流程。

## 11. 设计输入与影响

### TaskIntentDraft

为每个游戏实例提供游戏日志和 SourceMod 日志的安全只读查看能力。日志默认保留 14 天且全局可配置，在保留期内跨游戏、插件和 Overlay 重装持久存在。过期清理必须作为可观察的后台任务运行。查看器通过语义与 ANSI 高亮提升诊断效率。

### BaselineReadSetHint

现有实例使用 `instances/<id>/overlay/merged` 作为游戏容器挂载源，运行时写入进入实例 Overlay upper。生命周期服务已经区分游戏、私有文件、备份和控制台目录；持久任务系统已有串行、日志流、调度和系统设置基础。私有文件页面提供可复用的安全路径、两栏文件树和移动端抽屉交互，但日志服务不能复用其写入能力。

### ImpactStatementDraft

变更影响实例目录契约、容器挂载、升级迁移、生命周期与重装边界、日志读取服务、HTTP API、任务类型与调度、普通系统设置、实例导航和 React 日志查看器。兼容边界是保持现有内容所有权和任务串行语义；两个标准日志路径由新的实例持久日志目录接管。非目标是永久归档、日志编辑、实时跟随和统计分析。
