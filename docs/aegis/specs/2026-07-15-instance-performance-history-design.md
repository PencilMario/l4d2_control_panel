# 实例性能信息与一小时历史设计

- 状态：已批准
- 日期：2026-07-15
- 范围：总览实例卡片、实时资源采样、每实例网络统计与内存历史窗口

## 1. 目标

总览中的每个游戏实例应展示可定位当前负载问题的具体性能信息，而不只显示 CPU 和内存。指标包括 CPU、内存、网络、磁盘 I/O、进程数、本次运行时长和 A2S 响应延迟，并提供最近一小时的简单历史折线图。

网络同时展示当前下载/上传速率，以及本次实例运行累计接收/发送流量。累计值在实例每次启动时清零，停止后保留最后观测值，再次启动时开启新统计会话。

## 2. 非目标

- 不建立长期性能数据库，不在 Panel 重启后恢复历史。
- 不增加告警、容量预测、跨主机聚合或历史报表。
- 不提供数据包内容、地址、玩家身份或任意抓包查询。
- 不修改宿主机防火墙规则。
- 不统计实例未声明的动态端口。

## 3. 方案选择

采用把流量采集并入现有 `socket-proxy` 的方案，避免新增服务和镜像。代理进入 Host 网络，通过共享 Unix Socket 向 Panel 提供现有受限 Docker API和内部流量计数接口，并仅增加 `NET_RAW` 能力。

未采用独立采集容器，因为它会增加部署服务、镜像和内部 API。未采用按 cgroup 统计的 eBPF 方案，因为它会引入更高的内核兼容、权限和维护成本。

## 4. 安全与部署边界

`socket-proxy` 保持 Docker API 白名单，不向宿主机或 Compose 网络开放 TCP 监听。Panel 与代理通过只在两个控制容器间共享的 Unix Socket 通信。

代理使用 Host 网络以观察进入和离开宿主机网络接口、且属于实例声明端口的数据包。容器继续启用只读根文件系统、`no-new-privileges` 和 capability 最小化；在原有全部 capability 删除的基础上，仅恢复 `NET_RAW`。代理不保留原始数据包或连接元数据，只维护实例级收发字节计数。

Panel 向代理登记实例 ID、运行会话标识以及该实例拥有的游戏端口、可选 SourceTV 端口和已声明插件端口。代理只接受来自共享 Unix Socket 的登记和读取请求。

## 5. 数据采集

### 5.1 Docker 指标

扩展现有 Docker stats 解析，采集：

- CPU 使用率。
- 内存已用字节、限制字节和使用率。
- Block I/O 累计读取与写入字节。
- 当前进程数。

磁盘读写速率由相邻有效采样的累计值差除以实际时间间隔得到。首个样本、计数器回退、运行会话变化或异常采样间隔不生成速率。

### 5.2 网络指标

代理在非回环网络接口观察 IPv4 和 IPv6 的 TCP/UDP 流量，并按实例声明端口匹配：

- 目标端口属于实例时记为接收流量。
- 源端口属于实例时记为发送流量。
- 一个数据包同时匹配同一实例多个声明时只计一次。
- 不属于任何实例声明端口的数据包不统计。

代理为每个运行会话维护累计接收和发送字节。Panel 根据相邻样本计算当前下载和上传速率。Panel 自身发起的 A2S 健康查询属于实例端口流量，但相对游戏流量可忽略，且不另设排除规则。

### 5.3 运行时长与 A2S 延迟

本次运行时长以当前容器运行会话的开始时间为基准。A2S 延迟记录完成现有 INFO 查询的耗时；查询失败时延迟、地图和玩家摘要均为不可用，但不影响 Docker 或网络指标。

## 6. 采样与历史

Panel 统一负责每五秒采样所有运行实例，合并 Docker、流量代理和 A2S 数据。总览读取最近一次完成的观测，避免每个浏览器客户端重复触发采样。

每个实例使用容量为 720 的内存环形缓冲区，保存最近一小时采样。样本至少包含时间、运行会话标识、CPU 百分比、内存百分比、网络下载/上传速率及磁盘读取/写入速率。

实例停止或关键数据源不可用时写入相应空值。图表把空值渲染为断点，不连接不同运行会话。Panel 重启后历史窗口清空；SQLite 不承担性能时序数据所有权。

新实例或缓存尚未建立时，可立即触发一次受控采样，但相邻差值类速率必须等待第二个有效样本。

## 7. HTTP 契约

保留 `GET /api/instances/{id}/overview` 及现有字段，增加可空字段：

- 内存限制和内存百分比。
- 网络接收/发送速率及本次运行累计字节。
- 磁盘读取/写入速率及累计字节。
- 进程数、运行秒数和 A2S 延迟毫秒数。
- 当前运行会话标识或等价的不透明变化标记。

增加一个经过现有会话认证的只读历史接口，返回实例最近一小时的有序采样。历史响应不包含累计网络或磁盘总量，只包含绘图需要的速率和百分比，控制响应体大小。

新增字段遵循现有可空指标语义：数据不可用为 `null`，真实零值为 `0`。旧客户端可忽略新增字段；现有 `/resources`、`/players` 和生命周期接口保持兼容。

## 8. 总览界面

实例卡片保留状态、地图、玩家、版本、配置入口和快捷操作。性能区改为紧凑指标网格：

- CPU 当前百分比。
- 内存已用/限制及百分比。
- 网络当前下载/上传速率，小字显示本次运行累计收发。
- 磁盘当前读取/写入速率，小字显示累计读写。
- 进程数、本次运行时长和 A2S 延迟。

不可用值显示 `--`，真实零值保留为 `0`。字节与速率使用自动单位格式化，避免在窄屏溢出。

卡片下方放置固定高度的一小时折线图，使用 `recharts`。分段控制切换 `CPU`、`内存`、`网络`、`磁盘`：CPU 和内存各一条线；网络显示下载、上传两条线；磁盘显示读取、写入两条线。桌面端悬停、移动端点击可查看时间和精确值。停止和跨运行会话区间显示断线。

图表和指标网格必须保持稳定尺寸，不能因加载、空值、Tooltip 或最长单位文本造成卡片跳动或内容重叠。

## 9. 故障处理

- 流量采集失败时仅网络指标不可用，Docker 和 A2S 指标继续更新。
- Docker stats 失败时资源指标不可用，A2S 与上一次已冻结的累计网络值仍可分别表达自己的状态。
- A2S 超时只影响地图、玩家摘要和延迟。
- 代理重启、计数回退或运行会话变化时，Panel 丢弃该次速率差值，防止虚假峰值。
- 历史采样失败不阻塞生命周期操作，也不写入伪造零值。
- 未声明端口产生的流量明确不在统计范围内。

## 10. 测试策略

- Docker 单元测试：内存限制/百分比、Block I/O 聚合、进程数及可空边界。
- 流量采集单元与 Linux 集成测试：IPv4/IPv6、TCP/UDP、收发方向、端口归属、同实例去重、无关端口、会话重置和不保留数据包内容。
- 采样服务测试：五秒节拍、差值速率、首样本、计数回退、会话变化、部分数据源失败和 720 点覆盖。
- HTTP 测试：总览新增字段、真实零值与 `null`、历史顺序、认证和响应上限。
- React 测试：单位格式化、指标网格、图表模式切换、双序列、断点、Tooltip 和移动端布局。
- 回归验证：全部 Go 测试、`go vet`、前端单测、生产构建和 Playwright 总览流程。

由于 Windows 开发主机不能提供 Linux Host 网络和 `NET_RAW` 的真实抓包环境，协议归属逻辑使用可注入数据包测试；真实接口采集需要在一次性 Linux 部署主机执行 smoke test。

## 11. 兼容边界

保留 Host 网络、游戏实例非特权运行、Panel 不挂载 Docker Socket、Docker API 白名单、实例生命周期状态机、现有总览路由和五秒刷新体验。安全边界的唯一扩展是 `socket-proxy` 获得用于只读包计数的 `NET_RAW`，且其服务面从 TCP 收缩为共享 Unix Socket。

## 12. 设计输入与影响

### TaskIntentDraft

在总览为每个游戏实例展示更具体的实时性能信息，同时包含瞬时与累计带宽，并增加一个轻量的一小时历史图表。优先复用 `socket-proxy`，避免新增采集服务。

### BaselineReadSetHint

现有总览通过每实例 overview 接口合并 Docker/A2S 实时状态，Docker stats 当前只解析 CPU 和内存。所有游戏实例使用 Host 网络，因此 Docker 原生网络字段不能可靠表达每实例流量。`socket-proxy` 已是项目拥有的受限宿主机边界，但当前没有 Host 网络或网络读取能力。

### ImpactStatementDraft

变更影响 Docker stats 契约、socket-proxy 部署与安全能力、内部流量登记协议、Panel 后台采样与内存历史、HTTP JSON，以及 React 实例卡片和图表。兼容边界是保留现有生命周期、A2S、玩家、内容和更新行为；不引入长期监控系统或任意宿主机观测接口。
