# 实例永久删除设计

## 目标

在实例列表为每个游戏实例提供永久删除入口。删除操作必须移除实例记录、托管游戏容器和该实例的数据目录，且不可恢复。

## 现状

后端已提供 `DELETE /api/instances/{id}`，请求体支持 `confirm` 与 `delete_data`。生命周期服务会停止正在运行的实例、删除托管容器、删除实例记录，并在 `delete_data=true` 时删除 `<data-root>/instances/<id>`。前端实例卡片目前没有调用该能力的入口。

## 交互设计

实例卡片操作区增加危险操作按钮“删除实例”。点击后打开项目内确认弹窗，不使用浏览器原生确认。

弹窗展示实例名称，并明确说明以下内容会永久删除：

- 实例记录；
- 托管游戏容器；
- 该实例的数据目录，包括游戏文件、私有配置、插件、存档、备份和控制台数据。

管理员必须输入完整实例名称。仅当输入值与实例名称完全一致，且当前没有提交中的删除请求时，“永久删除”按钮才可用。取消或关闭弹窗不产生任何请求。

确认后前端发送：

```json
{
  "confirm": true,
  "delete_data": true
}
```

接口返回后台任务后，沿用现有任务状态与错误反馈。删除任务成功后关闭弹窗并刷新实例列表；请求失败或任务失败时保留实例卡片和弹窗上下文，显示错误且允许重试。

## 所有权与边界

后端生命周期服务继续作为停止容器、删除容器、删除数据库记录和清理目录的唯一所有者。前端不直接推断或执行分步清理，也不提供保留数据选项。

本次不改变现有删除 API、实例目录结构、任务持久化格式或任务日志保留策略。正在执行维护写入的实例继续由后端拒绝删除，前端展示后端返回的错误。

## 可访问性与响应式

删除按钮使用现有图标库和危险操作样式。确认弹窗包含明确标题、实例名称标签、文本输入、取消按钮和永久删除按钮；支持键盘焦点、Escape 关闭及移动端无横向溢出。按钮文案不得仅依赖颜色表达危险性。

## 验证

- 组件测试覆盖打开、取消、名称不匹配时禁用、名称匹配时提交固定永久删除请求、重复提交保护和错误保留。
- API/生命周期现有测试继续覆盖停止运行实例、删除容器、删除记录和按 `delete_data` 清理目录。
- Playwright 桌面与移动主流程覆盖确认弹窗、输入实例名、永久删除以及删除后实例从列表消失。

## 非目标

- 不支持仅删除记录或保留数据目录。
- 不支持批量删除实例。
- 不提供回收站、软删除或恢复能力。

## 设计输入

- TaskIntentDraft：为实例列表补充不可恢复的完整删除能力。
- BaselineReadSetHint：`internal/httpapi/server.go`、`internal/lifecycle/service.go`、`internal/store/store.go`、`web/src/app/App.tsx` 及现有生命周期测试。
- ImpactStatementDraft：前端实例卡片、确认弹窗、后台任务等待和实例列表刷新；后端契约保持不变。
