# 性能面板镜像大小设计

## 目标

在游戏实例性能面板的当前指标中移除重复的“玩家”，改为显示实例所用 Docker 镜像的本地大小；同时把网络历史图中的 `网络 RX`、`网络 TX` 改为更直观的“下载”、“上传”。

## 数据与职责

- Docker 适配层负责读取容器关联镜像的本地 `Size`，单位为字节。
- 性能采样快照新增可空字段 `image_size_bytes`。无法查询时保留为空，并沿用现有问题上报机制，不阻断其他指标。
- 总览性能接口返回 `image_size_bytes`，供当前指标展示。
- 性能历史接口不返回镜像大小。镜像大小变化频率低，不建立没有分析价值的历史曲线。

## 前端行为

- 性能当前指标中的“玩家”替换为“镜像大小”，使用现有 IEC 字节格式显示。
- 网络图表图例和提示框使用“下载”、“上传”。
- 总览卡片与玩家管理中的玩家数据保持不变。
- 镜像大小不可用时显示 `--`。

## 兼容边界

- 新 JSON 字段为可空增量字段，不改变现有字段含义。
- CPU、内存、网络、磁盘历史数据结构和采样窗口保持不变。
- 不增加镜像拉取、重建或磁盘清理行为。

## 验证

- Docker 适配层测试覆盖镜像大小读取和失败返回。
- 指标采样与 HTTP API 测试覆盖 `image_size_bytes` 的传播和空值。
- 前端测试确认玩家指标被移除、镜像大小被格式化，并确认网络文案为下载/上传。

## 工作草案

- TaskIntentDraft：以镜像大小替换性能面板中重复的玩家当前值，并简化网络术语。
- BaselineReadSetHint：`CONTEXT.md`、指标采样器、Docker 客户端、总览 API、`PerformancePanel` 及对应测试。
- ImpactStatementDraft：影响 Docker 读取、当前性能快照、总览 HTTP 契约和性能面板；不影响玩家功能、历史曲线种类或容器生命周期。
