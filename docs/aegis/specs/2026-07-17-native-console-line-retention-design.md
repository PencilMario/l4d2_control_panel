# 原生控制台行保留设计

## 目标

原生控制台在单个浏览器会话内仅保留并展示最新 1000 行输出，防止长时间连接导致前端内存和 DOM 文本持续增长。

## 现状与根因

前端当前保存最新 500 个 WebSocket 消息块。消息块与文本行没有一一对应关系：一个块可能包含多行，也可能只包含半行，因此该限制既不是 500 行，也无法稳定控制展示规模。

## 设计

前端以字符串作为控制台缓冲区。每次收到新的字符串或二进制 WebSocket 数据后追加文本，并按换行符裁剪：

- 最多保留最新 1000 行；
- 超出上限时从最旧完整行开始删除；
- 最后一段没有换行符的文本视为当前行并保留；
- 连续追加的半行必须正确合并；
- 不修改服务端 PTY/WebSocket 转发，不截断 SRCDS 自身输出。

控制台跟随逻辑继续以缓冲区版本触发滚动。关闭控制台后缓冲区随 React 组件销毁，不做跨会话持久化。

## 验证

- 单条消息包含超过 1000 行时仅保留最后 1000 行。
- 多条消息跨边界累计超过 1000 行时删除最旧行。
- 未换行的尾部内容在后续消息到达时正确拼接。
- 现有控制台自动跟随、向上滚动暂停和命令发送行为保持通过。

## 非目标

- 不增加服务端控制台日志持久化。
- 不改变 WebSocket 协议或 PTY 缓冲。
- 不提供用户可配置的行数上限。
