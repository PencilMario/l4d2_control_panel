# 最近任务日志与成功记录保留：实施意图

- 状态：planned
- 日期：2026-07-16
- 规格：../../specs/2026-07-16-recent-job-logs-retention-design.md

## 请求结果

管理员可以从最近任务列表原位展开每个任务的结构化生命周期日志，直接查看失败原因、发起时间、排队耗时和实际执行用时。系统全局默认只保留最近 25 个成功任务，并允许在系统设置中调整为 1 至 500。

## 变更类型

- SQLite 迁移与持久化契约扩展。
- Job Manager 生命周期事件记录。
- HTTP 详情与普通设置接口扩展。
- React 任务列表交互和系统设置表单。
- Store、Manager、HTTP、React 与 Playwright 回归测试。

## 风险提示

该任务跨越持久化、异步任务执行和用户界面，属于中等复杂度。主要风险是旧数据库迁移、任务快照与事件不一致、成功任务误删、现有轮询契约破坏和 SSE 响应膨胀。

## 非目标

不采集原始 stdout/stderr，不增加失败任务自动清理、日志搜索、下载、分页、任务重试或取消。
