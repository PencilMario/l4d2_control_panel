# Project Context: L4D2 Control Panel

## Language

| Term | Definition | Avoid |
|------|-----------|-------|
| 游戏实例 | 由 Panel 独立配置、运行和管理的一台 Left 4 Dead 2 专用服务器。 | 服务器进程、Docker 实例 |
| 总览观测 | Panel 为一个游戏实例采集的当前运行状态、玩家信息和性能信息快照。 | 数据库状态、历史统计 |
| 期望状态 | 管理员希望游戏实例保持的生命周期状态。 | 当前状态、实时状态 |
| 实际状态 | Panel 根据容器和游戏服务的实时响应判断出的游戏实例当前生命周期状态。 | 配置状态、期望状态 |
| 性能指标 | 从游戏实例当前运行环境采样得到的 CPU、内存、网络等资源使用数据。 | 配置参数、性能评分 |
| 当前带宽 | 一个游戏实例在采样区间内每秒接收和发送的数据量。 | 累计流量、网卡带宽上限 |
| 累计网络流量 | 一个游戏实例从本次运行开始累计接收和发送的数据量。 | 当前带宽、宿主机总流量 |
| 性能历史窗口 | Panel 在内存中为每个游戏实例保留的最近一小时性能采样序列。 | 长期监控、审计记录 |

## Relationships

- 一个 **游戏实例** 同时具有一个 **期望状态** 和一个 **实际状态**。
- 一次 **总览观测** 包含一个游戏实例的 **实际状态** 和可获得的 **性能指标**。
- **当前带宽** 由相邻采样间的 **累计网络流量** 差值计算。
- **性能历史窗口** 每五秒采样一次，并在 Panel 重启后清空。

## Flagged Ambiguities

- “实例状态”应明确为 **期望状态** 或 **实际状态**，避免混用。
